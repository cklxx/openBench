## Patterns

1. Experiment reports must be committed to git alongside the raw results data.
2. All documentation, memory, and comments must be in English.
3. Docs and memory structure follows the elephant.ai pattern: guides, memory, error/good experience, plans.
4. When simplifying code, fix all confirmed issues in one pass — do not ask per-issue.
5. README should contain only what is necessary to understand and use the project; no padding.
